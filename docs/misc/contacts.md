# 关于作者与路线图

NodeAuth 的成长离不开社区用户的反馈与支持。

---

## 📩 联系作者
如果您在使用过程中遇到问题、发现漏洞或有功能改进建议：
*   **GitHub Issues**：[NodeAuth 仓库](https://github.com/nodeauth/nodeauth-worker) (推荐)。
*   **Telegram 讨论组**：[NodeAuth Group](https://t.me/NodeauthGroup)。
*   **应急恢复包**：请参考 [./recovery.md](./recovery.md) 了解如何使用包含加密密钥的 PDF 文件进行账户恢复。

## 🗺️ 开发路线图 (Roadmap)
我们计划在未来的版本中加入以下特性：

- [ ] **开发原生 APP**：基于 Tauri/Rust 为桌面端及移动端提供更极致的性能与系统集成体验。
- [ ] **更多 OAuth 提供商**：加入如 GitLab, Bitbucket, Slack 等支持。
- [ ] **多语言扩展**：增加日文、俄文等支持。

## 🤝 共同开发 (Contributing)
NodeAuth 本着 100% 开源的精神，诚邀开发者加入我们的行列。根据 **开发路线图**，我们目前迫切需要以下领域的专才支持：

1.  **原生 APP 开发 (Tauri / Rust)**：
    *   **要求**：熟悉 Rust 语言及 Tauri 框架。
    *   **目标**：将 PWA 功能迁移至跨平台原生壳程序，整合更底层的系统通知与生物识别 API。
2.  **多语言社区支持 (i18n)**：
    *   **要求**：日文、俄文或德文等语言的母语使用者/专业翻译。
    *   **目标**：审核并纠正自动翻译中的专业词汇偏离。

如果您有意参与，请直接 [Telegram 讨论组](https://t.me/NodeauthGroup) 中联系我们。
