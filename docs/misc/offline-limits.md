# 离线功能边界说明

NodeAuth 的离线优先架构非常强大，但作为一款网络应用，部分功能在完全断网环境下是无法执行的。请提前知晓。

---

## ❌ 离线受限功能
当您开启了“离线模式”或处于无网络状态时，以下操作将暂时失效：

1.  **第三方 OAuth 登录**：无法通过 GitHub、Google 或 Telegram 完成初次登录验证。
2.  **云端同步分发**：本地的修改无法实时同步到其他设备。
3.  **异地备份上传**：无法向 S3、WebDAV 或 Telegram 机器人发送新的备份包。
4.  **远程会话注销**：无法强制踢出其他在线设备。
5.  **头像/图标实时拉取**：如果新添加的账号需要从云端实时搜索图标，离线时将无法获取预览。

## ✅ 离线可用功能
*   **查看验证码**：最核心功能，100% 离线计算。
*   **增删改账号**：操作会存入同步队列，待复网后自动重现。
*   **本地 JSON 导入/导出**：完全本地执行。
*   **搜索与分类**：全内存操作，响应极快。
*   **应用锁解锁**：PIN 码或生物识别均在本地验证。

> [!TIP]
> **最佳实践**：在环境不确定的情况下（如出国旅行前），请务必在有网环境下完成一次全量同步，并开启“离线就绪诊断”。
