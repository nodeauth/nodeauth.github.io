# 离线模式

离线模式是 NodeAuth 为对安全性有极致要求的极客准备的“物理隔离”功能。

---

## 🔒 什么是离线模式？
开启该模式后，NodeAuth 将进入 **Air-Gap 环境**：
*   **强制阻断**：所有发往服务器的 API 请求（包括同步、备份）会被应用层物理拦截。
*   **纯本地运行**：此时应用完全依赖本地 IndexedDB 中的冷解密数据工作。
*   **网络欺骗**：系统会向应用逻辑伪造一个“网络已断开”的状态，确保没有任何数据包流出。

## 🚜 适用场景
*   **出差/飞行**：在无网环境下通过 PWA 正常使用验证码。
*   **高压冷备份**：在环境不完全确定的公共网络下，开启离线模式进入，避免令牌泄露风险。
*   **隐私审查**：在此模式下，您可以安全地管理敏感账号。

## 🛠️ 如何开启与管理
1. 前往“系统设置” -> “**离线模式**”。
2. 系统会执行一次 **“离线就绪诊断”**，确认 WASM 核心和静态资源已完全预载（以防开启后由于资源缺失无法运行）。
3. 诊断通过后，点击蓝色的“启用”开关。
4. 首页顶部会出现显著的绿色“**已进入离线隔离状态**”通知。

> [!TIP]
> **注意**：离线模式下无法发起云端备份。执行关键操作后，建议在安全网络环境下短暂关闭该模式完成同步。
